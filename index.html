Welcome to BuddyPress template pack

The following is a guide to use when creating templates for the new BuddyPress template pack.

Resources

https://github.com/karmatosed/buddypress-templatepack *You will be given commit access to this once you have 2 pull requests accepted. This is done to ensure project consistency but still allow people to get things done.

In this repository is the skeleton for the plugin we’ll be using to create the template pack. <to come by end of weekend). This will allow you to try the templates without requiring a theme.

Github

You will either need to work with someone that knows how to do a pull request or ask for help with this before you can submit a template. We’re doing this to try and collect them in one place to start off.

Task process

  1. All tasks should be created as issues on Github or trac tickets
  2. Once you have been assigned a task you are agreeing to complete this. Only agree if you can see it through. If at any point you can’t or have an issue please let people know. Life happens, but we need to know.
  3. After you’ve given an estimate (see keeping everyone informed) you will then work on the task. At anytime if you need some help just ask - we’re all in this together.
  4. Once you’ve completed you can submit via a Github pull request.

Keeping everyone informed

  1. You will be asked to estimate the time for the task once you take it on. This estimate is important so we can get a gauge of things.
  2. If the task is going to take longer than a week then every Tuesday by midnight you need to update the task situation on the trac ticket or Github issue.

Submission of template process

  1. Work on assigned task (no task should be done without assignment of an issue on Github)
  2. Once done do a pull request for your changes.
  3. This will be either accepted or rejected with reasons so you can fix things.

HTML format

  1. We will be using html5
  2. When working out what element to use this flowchart works: http://html5doctor.com/downloads/h5d-sectioning-flowchart.pdf

CSS format

http://make.wordpress.org/core/handbook/coding-standards/css/

Additional rules:

  1. Ids should only be used for non styling - do not style on an id, use a class. http://screwlewse.com/2010/07/dont-use-id-selectors-in-css/
  2. Be wary of code smells : http://csswizardry.com/2012/11/code-smells-in-css/

CSS Files

  1. shame.css : this should be used for any hacks, !importants etc. The aim of this file is to be first in cleanup. N.B. please comment any filters or hacks used to show why the need for them so it can be followed up, but try to avoid using this file where possible.
  2. buddypress.css : this is where custom CSS will be added as each component is done.

Naming conventions

  1. Any BuddyPress will have an id “buddypress” around it (like we do currently).
  2. The file structure used in buddypress/ will be kept the same. Do not rename files. NOTE: this may get refactored later but we want to ensure we get templates done.
  3. Any id or class will be prefixed with bp- eg; bp-archive-members

Browsers and testing

Any template should be tested using the Twenty Thirteen theme so we’re all on the same page. The following browsers are also to be used for testing:

  • Safari mobile (iPhone)*
  • Safari desktop
  • Android*
  • Firefox (latest)
  • Opera (latest)
  • IE 8,9+
  • Chrome (latest)
  • iPad*
  • Macbook Air
  • 27”+ monitor *if you do not have mobile devices please say when take template
